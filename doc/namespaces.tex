\chapter{Namespaces}

% growth of code: so far most of the programs you have written has been small, maybe 10 lines of code (LoC), if you solve all the exercises of this book then you will write programs of more than 100 LoC, if you complete a software education you will likely participate if projects in excess of 1000 LoC, in the industry projects are often an order of magnitude or three larger, the linux kernel was in 2025 approx 40M LoC, most software projects grows over time as features are added and improvements are made

% clash of names: as the codebase grows so does the number and severity of problems stemming from that size, one of the very first problems you encounter is that of a name clash: you have found the right name for a variable or function and now you want to use the same name in a different context, the reasons are equally good but the compiler won't allow you to redefine the name, pure sadness, but what would it even mean? if you had two functions by the same name which implementation would be invoked when the name is called? given that there are two functions perhaps the choice of implementation should depend on where in the code the call site is located? and what about variables; when are they distinct and when are they the same? the situation appears to be a mess

% organization through namespaces: lets introduce some order to this chaos, up until now all the names that we have defined have existed within a single space, this is calles a namespace, conflicts in names are not allowed within a namespace, we can however have multiple namespaces and a name is allowed to be defined in multiple namespaces, when we refer to a name we either refer to the definition in the current namespace or we refer to it through the name of a different namespace, this mechanism allows us to name distict variables and functions without having to worry about whether they are used elsewhere, sure if it is used closeby there could be an issue but then you should ask yourself whether both names are expressive enough
